# Refactor — Безопасный рефакторинг

Рефакторинг кода без изменения поведения.

## Принципы

- Тесты должны оставаться зелёными на каждом шаге
- Маленькие атомарные изменения
- Отдельный коммит на каждый логический шаг

## Шаги

1. **Убедись тесты проходят** перед началом: `cargo test`
2. **Сделай изменение** (одно логическое)
3. **Проверь:** `cargo test && cargo clippy -- -D warnings`
4. **Коммит:** `refactor(scope): описание`
5. Повтори для следующего шага

## Запрещено при рефакторинге

- Менять крипто-примитивы (это не рефакторинг, это security change)
- Менять KDF-контексты (сломает все существующие ключи)
- Менять формат сериализации (сломает совместимость)
- Использовать `#[allow(clippy::...)]` как временный fix

## После рефакторинга

```bash
cargo fmt
cargo clippy -- -D warnings
cargo test
```
